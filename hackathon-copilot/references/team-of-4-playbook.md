# Team of 4 Playbook

## Role Split
- Builder A (Core backend/logic): data model, API/service reliability.
- Builder B (Frontend/demo UX): key flows, visuals, state handling.
- Integrator/DevOps: repo hygiene, deployment, CI, env, backup demo path.
- PM/Pitch lead (can also code): backlog control, acceptance checks, deck, narrative, judge Q&A prep.

## Timeline (24-36h style)
- Hour 0-1: idea lock + architecture sketch + acceptance criteria.
- Hour 1-3: scaffold app + one end-to-end thin vertical slice.
- Hour 3-8: feature sprint 1 (core loop), ship testable build.
- Hour 8-14: feature sprint 2 (differentiator), tighten UX.
- Hour 14-18: polish + instrumentation + bug burn.
- Final 3-4h: freeze code, rehearse demo, prep backup assets.

## Handoff Standard
Every PR/update includes:
1. what changed
2. what can break
3. how to test in 1 minute

## Standup Cadence
Every 90 minutes:
- Done since last check
- Next 90-minute target
- Blockers (owner + ETA)
