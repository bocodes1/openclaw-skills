# Idea Scorecard (Hackathon)

Score each idea 1-5 in each category, then multiply by weight.

## Categories
- Judge clarity (x3): Can judges understand value in 20 seconds?
- Demo wow (x3): Is there a visible, memorable moment?
- Build feasibility (x3): Can team ship credible MVP in available time?
- Technical signal (x2): Does implementation look non-trivial and intentional?
- User impact/story (x2): Is the pain real and story emotionally clear?
- Risk profile (x2): Lower dependency risk = higher score.

## Selection Rule
Pick the highest weighted score, then run this veto check:
- Requires >2 unknown integrations? reject.
- Cannot demo core value without internet? reject unless strong fallback.
- Cannot explain in one sentence? reject.

## One-Sentence Thesis Template
"For [specific user], who struggles with [pain], we built [product] that [core outcome] using [technical differentiator]."
