# Prompt Pack (Use During Hackathon)

## 1) Pick the Best Idea
"We are a team of 4 in a hackathon with [time], [theme], [constraints]. Generate 5 ideas, then score them using: judge clarity, demo wow, feasibility, technical signal, impact, risk. Return ranked table + top pick + 3 reasons."

## 2) Scope Ruthlessly
"Given this idea: [idea], define MVP for 1-day hackathon. Output must-have vs nice-to-have, and hard cuts that protect submission quality."

## 3) Build Plan
"Create a 12-hour build plan for a 4-person team. Include parallel workstreams, dependencies, and checkpoint times every 90 minutes."

## 4) Judge-Focused Pitch
"Turn this project into a 2-minute pitch: hook, pain, solution, demo flow, technical depth, impact, close. Keep it high energy and concrete."

## 5) Q&A Defense
"Act as strict hackathon judges. Ask 15 hard questions about novelty, feasibility, privacy/security, and competition. Then provide concise winning answers."

## 6) Last-Minute Rescue
"We have [X] hours left and these blockers: [list]. Propose a rescue plan that maximizes judging score with minimal risky code changes."
