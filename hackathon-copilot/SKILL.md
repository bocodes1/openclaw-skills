# Hackathon Copilot

## Overview
Run a practical win-focused workflow: pick a judge-friendly idea, build only what scores, and ship a polished demo story under time pressure.

## Quick Workflow
1. Define constraints in 5 minutes: theme, hard requirements, team size, deadline, allowed stack.
2. Score idea options against judging criteria using `references/idea-scorecard.md`.
3. Lock an MVP slice that can be demoed in under 2 minutes.
4. Split roles and deliverables using `references/team-of-4-playbook.md`.
5. Build in short cycles (90-minute sprints) with regular risk checks.
6. Prepare pitch, demo script, and fallback plan 3-4 hours before submission.

## Execution Rules
- Optimize for judged outcomes, not technical purity.
- Prefer one unforgettable feature over five average ones.
- Freeze scope early; only add features that improve judged criteria.
- Keep an offline/screenshot fallback in case demo environment fails.
- Maintain a visible "done list" and "risk list" throughout.

## Deliverables to Produce
- One-line project thesis: user + pain + magic + why now.
- 3-bullet judging mapping: innovation, technical depth, impact/UX.
- MVP checklist (must-have vs nice-to-have).
- Demo flow script (90-120 seconds).
- Q&A bank for likely judge questions.

## If Time Is Collapsing
- Cut integrations first, keep core user loop intact.
- Replace flaky live APIs with cached/sample data.
- Record a 45-second backup demo video.
- Shift one teammate fully to polish (UX text, visual consistency, onboarding).
